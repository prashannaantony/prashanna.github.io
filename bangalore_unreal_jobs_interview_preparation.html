Core areas to prepare

C++ fundamentals
Pointers, memory management, OOP, RAII, templates — Unreal uses heavy C++
Blueprints scripting
Visual scripting system — know event graphs, variables, functions, macros
UE5 architecture
World, GameMode, GameState, PlayerController, Pawn, Actor lifecycle
Performance & optimization
LODs, culling, Nanite, Lumen, profiling tools (Unreal Insights)
Game math
Vectors, quaternions, matrices, collision math, physics basics
Version control
Git + Perforce (used at most studios) — branching, merging, conflict resolution

Common interview questions

  • Explain UObject vs AActor vs UActorComponent — when do you use each?
  • How does Unreal's garbage collection work? What is TObjectPtr?
  • Difference between a Blueprint interface and a C++ interface
  • How would you optimize a game running at low FPS? Walk through your process
  • Explain the replication system for multiplayer in Unreal
  • What is the difference between Tick() and Timer-based updates?
  • Describe a bug you fixed — how did you debug and isolate it?
Build a small demo project in UE5 before your walk-in — even a simple third-person prototype shows initiative. Bring a laptop or video recording.

Walk-in interview checklist

  • Printed CV (5–6 copies) — keep it to 1–2 pages, highlight UE5/C++ projects
  • Passport-size photos (2 copies) — some studios ask for ID docs
  • Govt ID proof (Aadhaar / PAN card)
  • Laptop with your demo project / portfolio loaded and ready
  • Portfolio links (GitHub, ArtStation, itch.io) bookmarked and accessible
  • Studio address confirmed — Bangalore traffic is heavy, arrive 30 min early
  • Walk-in timing checked — most are 10 AM–1 PM; call ahead to confirm
  • Pen and notepad for on-spot coding/design rounds
  • Smart-casual dress — game studios are informal but first impressions matter
  • Expected salary range prepared — research Bangalore UE dev salaries on Glassdoor
